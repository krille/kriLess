Utility classes

Signify that this class is a Utility class. It has a very specific role (often providing only one declaration) and should not be bound onto or changed. It can be reused and is not tied to any specific piece of UI.

Prefix: .u-

Spacing

Margin and padding helper classes. Use these to tweak layout on a micro level.

.u-(m|p)(t|r|b|l|h|v)(-|+|0) {} = margin/padding
top/right/bottom/left/horizontal/vertical less/more/none

Works with mq-prefixes e.g. .u-lap-mb+, .u-desk-mb++ 

<div class="u-p u-pv+ u-mb u-lap-and-up-mb+"></div>

Widths

Width helper classes. Use these to tweak layout on a micro level.

.u-(one-half|1/2) {} = width: 50%

Works with mq-prefixes e.g. .u-lap-and-up-1/2, .u-desk-1/3 

<div class="o-layout__item u-desk-1/2"></div>

Mixins

A number of default mixins that the framework, and developers working with it, can make use of.

.font-size(@font-size, @line-height)

Params:

{int} @font-size - can be any integer or a pixel unit e.g. 16px,
{int} [@line-height] - (optional) can be any integer or a keyword e.g. inherit, or normal

Description:

Sets the font-size property with both px and rem units.
Optionally with a specified line-height or no line-height at all.

If no line-height is specified the mixin will calculate an appropriate one for you.

Example 1: Fully formed type style (sizing and vertical rhythm) from a single value

	.foo { 
		.font-size(12px); 
	}

This will yield:

	.foo {
		font-size: 12px;
		font-size: 0.75rem;
		line-height: 2;
	}

Example 2: Font-size with a specific line-height

	.foo { 
		.font-size(12px, 1.5);
	}

This will yield:

	.foo {
		font-size: 12px;
		font-size: 0.75rem;
		line-height: 1.5;
	}

Example 3: Font-size with no line-height at all

	.foo { 
		.font-size(12px, none); 
	}

This will yield:

	.foo {
		font-size: 12px;
		font-size: 0.75rem;
	}
